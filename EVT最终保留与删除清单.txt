CH32H417 无人机项目：EVT 最终保留与删除清单

说明：
这份清单是结合你们当前目标和你最新补充的外设信息整理的：
- 匿名光流
- 六轴 IMU
- TOF 测距
- 气压计（由其他芯片管理）
- GPS
- NRF 遥控
- 视觉识别由另一颗芯片处理
- 电磁铁投放

目标是：
1. 保留足够支撑飞控开发的资料
2. 尽量缩小 GitHub 仓库体积
3. 避免删掉后面大概率还要回头找的例程

====================
一、建议“必须保留”的目录
====================

1. EXAM/SRC
原因：
- 底层标准外设库
- 启动文件
- 链接脚本
- Debug/Delay/基础支持
这是整个工程能编起来的根基，必须保留。

2. EXAM/TIM
原因：
- 电机 PWM 输出要用
- 定时器中断、捕获、同步等也可能会参考
重点看：
- TIM/PWM_Output
- TIM/TIM_INT
- TIM/Input_Capture

3. EXAM/USART
原因：
- 串口 IMU
- GPS
- 视觉板通信
- 调试串口
重点看：
- USART_Interrupt
- USART_DMA
- USART_Polling

4. EXAM/SPI
原因：
- NRF24L01 常见用 SPI
重点看：
- 2Lines_FullDuplex
- SPI_DMA

5. EXAM/GPIO
原因：
- 电磁铁控制
- LED 状态指示
- 基础引脚控制
- 你当前自己的调试工程也在这里

6. EXAM/EXTI
原因：
- NRF 中断
- 传感器中断
- 外部事件触发

7. EXAM/DMA
原因：
- 串口 DMA
- SPI DMA
- 减轻主循环负担

8. EXAM/ADC
原因：
- 建议后面接电池电压采样
- 如果要做电流采样，也会用到

9. EXAM/RCC
原因：
- 时钟配置参考
- 飞控项目对定时和采样频率非常敏感

10. EXAM/PWR
原因：
- 低功耗不一定是重点
- 但电源管理、欠压相关逻辑仍有参考价值

11. EXAM/IWDG
12. EXAM/WWDG
原因：
- 比赛项目强烈建议做看门狗保护

13. EXAM/FLASH
原因：
- 可保存校准参数、PID 参数、配置项

14. EXAM/I2C
原因：
- 你们已经有 TOF、气压计，后面还很可能加磁力计
- 所以 I2C 不是“可留可不留”，而是建议明确保留

====================
二、建议“有条件保留”的目录
====================

1. EXAM/CPU
保留条件：
- 如果你们还准备继续探索双核（V3F + V5F）
保留原因：
- HSEM
- IPC
- SYSTICK
- Core 行为差异

如果后面你决定整个飞控都先收敛到 V3F 单核，这个目录的重要性会下降，但我仍建议短期内先保留。

2. EXAM/CAN
保留条件：
- 如果以后要接 CAN 外设（例如某些电调、扩展模块）
否则当前可以不保留。

3. EXAM/RTC
保留条件：
- 如果你们后面做日志时间戳、长时任务或者掉电时间记录
否则当前不是刚需。

4. EXAM/CRC
保留条件：
- 如果你们想把跨芯片通信协议做得更稳，后面可能会用 CRC 做数据帧校验
- 但这不是必须依赖 EVT 例程才能做

====================
三、建议“可以删除或不上传 GitHub”的目录
====================

下面这些按你们当前无人机项目，大概率不需要：

1. EXAM/ETH
- 以太网
- 当前飞控项目基本不用
- 而且体积较大

2. EXAM/LTDC
- LCD 显示控制器
- 你们当前没有本地显示界面需求

3. EXAM/FMC
- 外扩 SDRAM/NAND/NOR/LCD 总线
- 小型飞控项目基本不需要

4. EXAM/SDIO
5. EXAM/SDMMC
- 如果你们不在 H417 上挂 SD 卡 / eMMC 做日志，就可以删

6. EXAM/DVP
- 摄像头并口接口
- 你们的视觉已经放在另一颗芯片上处理

7. EXAM/SAI
8. EXAM/I2S
- 音频接口
- 当前项目无关

9. EXAM/UHSIF
10. EXAM/SerDes
11. EXAM/SWPMI
12. EXAM/ECDC
13. EXAM/GPHA
- 明显偏专用高速接口/图形接口
- 当前飞控项目不相关

14. EXAM/TKey
- 触摸按键
- 用不到

15. EXAM/DFSDM
- 专用滤波/采样接口
- 当前不用

16. EXAM/HSADC
- 高速 ADC
- 当前阶段不需要

17. EXAM/OPA
- 片上运放/比较器
- 当前大概率不用

18. EXAM/DAC
- 数模输出
- 当前飞控链路里通常不用

19. EXAM/I3C
- 当前项目几乎不会优先采用

20. EXAM/IAP
- 如果你们不做升级功能，可删
- 体积也不算小

21. EXAM/PIOC
- 专用 IO 协处理相关
- 当前不需要

22. EXAM/APPLICATION
- 当前只有 SoftUART 一类示例
- 对你们现有链路不是关键

====================
四、建议“暂时不要急着删”的目录
====================

1. EXAM/ADC
- 因为电池采样强烈建议后面补

2. EXAM/I2C
- 因为 TOF / 气压计 / 未来磁力计都高度相关

3. EXAM/CPU
- 因为你现在还在双核平台上摸索

4. EXAM/TIM
- 不只是 PWM，后面调度、中断、输入捕获也可能要看

5. EXAM/USART
- 你们系统高度依赖串口链路

====================
六、根目录中建议不上传 GitHub 的内容
====================

1. PUB/
原因：
- 更像资料或分发内容，不是主工程源码

2. 论文合集/
原因：
- 明显不是固件工程源码的一部分

3. CH32H417_List.txt
4. CH32H417_List_EN.txt
原因：
- 清单资料可本地保留，不必进代码仓库

====================
七、我最推荐的仓库整理方式
====================

不建议你直接在原 EVT 上做“大规模不可逆删除”，更推荐下面两种方式之一：

方案 A：新建你们自己的飞控仓库（最推荐）
- 新仓库只放：
  1）你们自己的主工程
  2）EXAM/SRC
  3）你们实际参考的少量例程
- 原始 EVT 整包留在本地备份

优点：
- 仓库最干净
- 后期协作最清楚
- 不怕哪天想回头翻官方例程时找不到

方案 B：在当前 EVT 基础上裁剪
- 先把上面“必须保留”的目录单独备份
- 再删“可以删除”的目录

优点：
- 操作直接
缺点：
- 后期可能会后悔删得太早
- 原仓库历史会比较混乱

====================
八、最终一句建议
====================

如果你问我最稳的做法：
- 不要把整个 EVT 当成你们长期 GitHub 主仓库。
- 最好是新建一个干净的无人机飞控仓库，然后把真正需要的东西迁进去。

这样最适合比赛项目持续开发。
