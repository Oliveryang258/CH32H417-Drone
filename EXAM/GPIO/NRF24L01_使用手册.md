## 1. NRF24L01简介

### 1.1 什么是NRF24L01？
NRF24L01是一款工作在2.4GHz ISM频段的无线收发芯片，具有以下特点：
- **工作频率**：2.4~2.525 GHz（126个频道，每个频道间隔1MHz）
- **数据速率**：250Kbps、1Mbps、2Mbps可选
- **发射功率**：-18dBm、-12dBm、-6dBm、0dBm可选
- **通信距离**：开阔地带最远可达100米（0dBm功率）
- **接口方式**：SPI通信（最高10MHz）
- **自动应答**：支持硬件自动ACK，无需软件干预
- **多通道接收**：支持6个接收通道（Pipe0~Pipe5）

### 1.2 应用场景
- 无线遥控器
- 无线传感器网络
- 无线数据传输
- 游戏手柄
- 智能家居控制

## 2. 通信原理

### 2.1 基本通信模型
NRF24L01采用**主从模式**通信，一个设备发送，另一个设备接收。

```
设备A (发送端)                    设备B (接收端)
    |                                  |
    |  1. 切换到TX模式                  |  1. 切换到RX模式
    |  2. 写入数据到TX FIFO            |  2. CE保持高电平，持续监听
    |  3. CE拉高10us触发发送            |
    |  4. 发送数据包 ----------------> |  3. 接收到数据
    |                                  |  4. 自动发送ACK应答
    |  5. 接收ACK <------------------- |
    |  6. TX_DS标志置位（发送成功）     |  5. RX_DR标志置位（接收成功）
```

### 2.2 地址机制（重要！）
NRF24L01使用**地址**来识别通信对象，类似于网络中的IP地址。

#### 关键概念：
- **TX_ADDR**：发送目标地址（我要发给谁）
- **RX_ADDR_P0**：接收ACK的地址（必须与TX_ADDR相同）
- **RX_ADDR_P1**：接收数据的地址（我的地址）

#### 配置示例：
假设有两个设备A和B：

**设备A的配置：**
```c
TX_ADDR      = 0x34 43 10 10 B1  // 发送给B
RX_ADDR_P0   = 0x34 43 10 10 B1  // 接收B的ACK
RX_ADDR_P1   = 0x34 43 10 10 A1  // 我的地址是A
```

**设备B的配置：**
```c
TX_ADDR      = 0x34 43 10 10 A1  // 发送给A
RX_ADDR_P0   = 0x34 43 10 10 A1  // 接收A的ACK
RX_ADDR_P1   = 0x34 43 10 10 B1  // 我的地址是B
```

### 2.3 自动应答机制
NRF24L01支持硬件自动ACK：
1. 发送端发送数据后，自动切换到RX模式等待ACK
2. 接收端收到数据后，自动发送ACK应答
3. 如果发送端未收到ACK，会自动重传（最多重传ARC次）
4. 重传间隔为ARD（默认500us）

### 2.4 数据包格式
每个数据包包含：
- **前导码**（1字节）：自动添加
- **地址**（3~5字节）：目标地址
- **载荷**（1~32字节）：用户数据
- **CRC校验**（1~2字节）：自动添加

## 3. 硬件连接

### 3.1 引脚定义
```
NRF24L01模块    ->    MCU引脚
---------------------------------
VCC (3.3V)      ->    3.3V电源
GND             ->    GND
CE              ->    PD1 (GPIO输出)
CSN             ->    PD0 (GPIO输出)
SCK             ->    PC10 (SPI3_SCK)
MOSI            ->    PC12 (SPI3_MOSI)
MISO            ->    PC11 (SPI3_MISO)
IRQ             ->    PD2 (GPIO输入，可选)
```

### 3.2 注意事项
- **电源**：NRF24L01工作电压为1.9~3.6V，建议使用3.3V
- **电流**：发送时峰值电流可达13.5mA，建议加100uF电容
- **天线**：模块自带PCB天线或外接天线
- **距离**：开阔环境下，0dBm功率可达50~100米

## 4. 代码架构

### 4.1 文件结构
```
bsp_nrf.h       - 头文件，定义接口和数据结构
bsp_nrf.c       - 实现文件，包含所有驱动函数
board_config.h  - 板级配置，定义引脚
```

### 4.2 API分层
```
应用层
  ├─ Ping/Pong测试层（高级封装）
  │   ├─ NRF_PP_InitInitiator()    // 初始化为发起端
  │   ├─ NRF_PP_InitResponder()    // 初始化为响应端
  │   └─ NRF_PP_Process()          // 轮询处理
  │
  ├─ 通用通信层（中级接口）
  │   ├─ NRF_Config()              // 配置NRF参数
  │   ├─ NRF_Transmit()            // 发送数据
  │   ├─ NRF_ReadRXPayload()       // 接收数据
  │   ├─ NRF_SetMode_TX()          // 切换到发送模式
  │   └─ NRF_SetMode_RX()          // 切换到接收模式
  │
  └─ 底层驱动层（寄存器操作）
      ├─ NRF_Init()                // 初始化硬件
      ├─ NRF_Check()               // 检查模块在线
      ├─ NRF_ReadReg()             // 读寄存器
      └─ NRF_WriteReg()            // 写寄存器
```

## 5. 快速开始

### 5.1 最简单的方式：使用Ping/Pong测试

#### 设备A（发起端）代码：
```c
#include "bsp_nrf.h"

int main(void)
{
    // 系统初始化
    SystemInit();
    Delay_Init();
    USART_Printf_Init(115200);
    
    // 初始化为发起端（会自动发送ping，等待pong）
    if (NRF_PP_InitInitiator() != NRF_OK) {
        printf("NRF初始化失败！\r\n");
        while(1);
    }
    
    printf("NRF初始化成功，开始发送ping...\r\n");
    
    // 主循环
    while(1) {
        NRF_PP_Process();  // 处理ping/pong通信
        
        // 可选：打印统计信息
        const NRF_PP_Stats_t *stats = NRF_PP_GetStats();
        printf("发送成功:%lu 失败:%lu 收到pong:%lu\r\n",
               stats->tx_ok_count,
               stats->tx_fail_count,
               stats->pong_ok_count);
        
        Delay_Ms(1000);
    }
}
```

#### 设备B（响应端）代码：
```c
#include "bsp_nrf.h"

int main(void)
{
    // 系统初始化
    SystemInit();
    Delay_Init();
    USART_Printf_Init(115200);
    
    // 初始化为响应端（会持续接收，收到ping后自动回复pong）
    if (NRF_PP_InitResponder() != NRF_OK) {
        printf("NRF初始化失败！\r\n");
        while(1);
    }
    
    printf("NRF初始化成功，等待接收ping...\r\n");
    
    // 主循环
    while(1) {
        NRF_PP_Process();  // 处理ping/pong通信
        
        // 可选：打印统计信息
        const NRF_PP_Stats_t *stats = NRF_PP_GetStats();
        printf("接收成功:%lu 发送pong:%lu\r\n",
               stats->rx_ok_count,
               stats->tx_ok_count);
        
        Delay_Ms(1000);
    }
}
```

### 5.2 运行结果
如果一切正常，你会看到：
- **设备A**：每秒发送一个ping，并收到pong应答
- **设备B**：接收到ping，自动回复pong

## 6. 进阶使用

### 6.1 自定义通信协议

如果Ping/Pong测试不满足需求，可以使用通用通信层API。

#### 6.1.1 发送端示例
```c
#include "bsp_nrf.h"

int main(void)
{
    NRF_Config_t cfg;
    uint8_t tx_buf[32];
    uint8_t local_addr[5] = {0x34, 0x43, 0x10, 0x10, 0xA1};  // 我的地址
    uint8_t peer_addr[5]  = {0x34, 0x43, 0x10, 0x10, 0xB1};  // 对端地址
    
    // 系统初始化
    SystemInit();
    Delay_Init();
    USART_Printf_Init(115200);
    
    // 初始化NRF硬件
    NRF_Init();
    
    // 检查模块是否在线
    if (NRF_Check() != 1) {
        printf("NRF模块未检测到！\r\n");
        while(1);
    }
    
    // 配置NRF参数
    cfg.mode = NRF_MODE_TX;              // TX模式
    cfg.channel = 40;                    // 频道40 (2440MHz)
    cfg.data_rate = NRF_DR_1MBPS;        // 1Mbps速率
    cfg.tx_power = NRF_PWR_0DBM;         // 0dBm功率
    cfg.local_addr = local_addr;         // 本机地址
    cfg.peer_addr = peer_addr;           // 对端地址
    cfg.addr_width = 5;                  // 5字节地址
    cfg.payload_width = 32;              // 32字节载荷
    
    if (NRF_Config(&cfg) != NRF_OK) {
        printf("NRF配置失败！\r\n");
        while(1);
    }
    
    printf("NRF配置成功，开始发送数据...\r\n");
    
    // 主循环：发送数据
    uint32_t counter = 0;
    while(1) {
        // 准备数据
        sprintf((char*)tx_buf, "Hello NRF! Count=%lu", counter++);
        
        // 发送数据
        NRF_Status_t status = NRF_Transmit(tx_buf, 32, 100);
        
        if (status == NRF_OK) {
            printf("发送成功：%s\r\n", tx_buf);
        } else if (status == NRF_ERROR) {
            printf("发送失败：未收到ACK\r\n");
        } else {
            printf("发送超时\r\n");
        }
        
        Delay_Ms(1000);
    }
}
```

#### 6.1.2 接收端示例
```c
#include "bsp_nrf.h"

int main(void)
{
    NRF_Config_t cfg;
    uint8_t rx_buf[32];
    uint8_t local_addr[5] = {0x34, 0x43, 0x10, 0x10, 0xB1};  // 我的地址
    uint8_t peer_addr[5]  = {0x34, 0x43, 0x10, 0x10, 0xA1};  // 对端地址
    
    // 系统初始化
    SystemInit();
    Delay_Init();
    USART_Printf_Init(115200);
    
    // 初始化NRF硬件
    NRF_Init();
    
    // 检查模块是否在线
    if (NRF_Check() != 1) {
        printf("NRF模块未检测到！\r\n");
        while(1);
    }
    
    // 配置NRF参数（注意：local_addr和peer_addr与发送端相反）
    cfg.mode = NRF_MODE_RX;              // RX模式
    cfg.channel = 40;                    // 频道40（必须与发送端一致）
    cfg.data_rate = NRF_DR_1MBPS;        // 1Mbps速率（必须与发送端一致）
    cfg.tx_power = NRF_PWR_0DBM;         // 功率（接收时不重要）
    cfg.local_addr = local_addr;         // 本机地址
    cfg.peer_addr = peer_addr;           // 对端地址
    cfg.addr_width = 5;                  // 5字节地址（必须与发送端一致）
    cfg.payload_width = 32;              // 32字节载荷（必须与发送端一致）
    
    if (NRF_Config(&cfg) != NRF_OK) {
        printf("NRF配置失败！\r\n");
        while(1);
    }
    
    printf("NRF配置成功，等待接收数据...\r\n");
    
    // 主循环：接收数据
    while(1) {
        // 检查是否有数据
        if (NRF_DataReady()) {
            // 读取数据
            uint8_t len = NRF_ReadRXPayload(rx_buf, 32);
            
            if (len > 0) {
                rx_buf[31] = '\0';  // 确保字符串结束
                printf("接收到数据：%s\r\n", rx_buf);
                
                // 清除RX_DR中断标志
                NRF_Clear_RX_DR();
            }
        }
        
        Delay_Ms(10);  // 轮询间隔
    }
}
```

### 6.2 双向通信

如果需要双向通信（A发送给B，B也发送给A），需要在接收到数据后切换模式。

#### 示例：设备A（主动发送，被动接收）
```c
while(1) {
    // 1. 发送数据
    sprintf((char*)tx_buf, "A->B: %lu", counter++);
    NRF_SetMode_TX();
    NRF_Transmit(tx_buf, 32, 100);
    
    // 2. 切换到接收模式，等待B的回复
    NRF_SetMode_RX();
    Delay_Ms(100);  // 等待B发送
    
    if (NRF_DataReady()) {
        NRF_ReadRXPayload(rx_buf, 32);
        printf("收到B的回复：%s\r\n", rx_buf);
        NRF_Clear_RX_DR();
    }
    
    Delay_Ms(1000);
}
```

#### 示例：设备B（被动接收，主动回复）
```c
while(1) {
    // 1. 接收A的数据
    if (NRF_DataReady()) {
        NRF_ReadRXPayload(rx_buf, 32);
        printf("收到A的数据：%s\r\n", rx_buf);
        NRF_Clear_RX_DR();
        
        // 2. 回复A
        sprintf((char*)tx_buf, "B->A: ACK");
        NRF_SetMode_TX();
        NRF_Transmit(tx_buf, 32, 100);
        NRF_SetMode_RX();  // 切回接收模式
    }
    
    Delay_Ms(10);
}
```

## 7. 协议设计指南

### 7.1 数据包格式设计

建议的数据包格式：
```c
typedef struct {
    uint8_t  magic;        // 魔数，用于识别数据包（如0xA5）
    uint8_t  type;         // 数据包类型（如0x01=命令，0x02=数据）
    uint8_t  sender_id;    // 发送者ID
    uint8_t  seq;          // 序列号（用于检测丢包）
    uint8_t  payload[26];  // 有效载荷
    uint8_t  checksum;     // 校验和（简单的XOR校验）
} MyPacket_t;  // 总共32字节
```

### 7.2 校验和计算
```c
uint8_t CalculateChecksum(const uint8_t *data, uint8_t len)
{
    uint8_t checksum = 0;
    for (uint8_t i = 0; i < len; i++) {
        checksum ^= data[i];
    }
    return checksum;
}
```

### 7.3 数据包验证
```c
uint8_t ValidatePacket(const MyPacket_t *pkt)
{
    // 1. 检查魔数
    if (pkt->magic != 0xA5) {
        return 0;
    }
    
    // 2. 检查校验和
    uint8_t calc_checksum = CalculateChecksum((uint8_t*)pkt, 31);
    if (calc_checksum != pkt->checksum) {
        return 0;
    }
    
    return 1;  // 数据包有效
}
```

### 7.4 协议示例：遥控器

#### 遥控器端（发送）
```c
typedef struct {
    uint8_t  magic;        // 0xA5
    uint8_t  type;         // 0x01=摇杆数据
    uint8_t  sender_id;    // 遥控器ID
    uint8_t  seq;          // 序列号
    int16_t  joystick_x;   // 摇杆X轴（-100~100）
    int16_t  joystick_y;   // 摇杆Y轴（-100~100）
    uint8_t  buttons;      // 按键状态（位域）
    uint8_t  reserved[21]; // 保留
    uint8_t  checksum;     // 校验和
} RemotePacket_t;

// 发送函数
void SendRemoteData(int16_t x, int16_t y, uint8_t buttons)
{
    static uint8_t seq = 0;
    RemotePacket_t pkt;
    
    pkt.magic = 0xA5;
    pkt.type = 0x01;
    pkt.sender_id = 0x01;
    pkt.seq = seq++;
    pkt.joystick_x = x;
    pkt.joystick_y = y;
    pkt.buttons = buttons;
    memset(pkt.reserved, 0, 21);
    pkt.checksum = CalculateChecksum((uint8_t*)&pkt, 31);
    
    NRF_Transmit((uint8_t*)&pkt, sizeof(pkt), 100);
}
```

#### 小车端（接收）
```c
void ProcessRemoteData(void)
{
    RemotePacket_t pkt;

        // 验证数据包

            // 控制电机
            SetMotorSpeed(pkt.joystick_x, pkt.joystick_y);
            
            // 处理按键
            if (pkt.buttons & 0x01) {
                // 按键1按下
            }
        }
    }
}
```

## 8. 常见问题

### 8.1 NRF_Check()返回0，模块检测失败

**可能原因：**
1. **供电问题**：检查3.3V电源是否正常
2. **接线错误**：
   - MOSI和MISO接反
   - CSN未连接或接错
3. **SPI配置错误**：检查CPOL和CPHA是否为0
4. **模块损坏**：更换模块测试

**调试方法：**
```c
NRF_Init();
NRF_Diagnose();  // 打印详细诊断信息
```

### 8.2 发送失败，NRF_Transmit()返回NRF_ERROR

**可能原因：**
1. **接收端未开启**：确保接收端处于RX模式
2. **地址配置错误**：
   - 发送端的TX_ADDR必须等于接收端的RX_ADDR_P1
   - 发送端的RX_ADDR_P0必须等于TX_ADDR（用于接收ACK）
3. **频道不一致**：发送端和接收端的channel必须相同
4. **数据率不一致**：发送端和接收端的data_rate必须相同
5. **距离过远或有遮挡**：尝试缩短距离或提高发射功率

**调试方法：**
```c
// 打印配置信息
printf("TX_ADDR: %02X %02X %02X %02X %02X\r\n", ...);
printf("RX_ADDR_P0: %02X %02X %02X %02X %02X\r\n", ...);
printf("RX_ADDR_P1: %02X %02X %02X %02X %02X\r\n", ...);
printf("Channel: %u\r\n", cfg.channel);
```

### 8.3 接收不到数据，NRF_DataReady()始终返回0

**可能原因：**
1. **地址配置错误**：接收端的RX_ADDR_P1必须等于发送端的TX_ADDR
2. **未切换到RX模式**：确保调用了NRF_SetMode_RX()
3. **CE引脚未拉高**：RX模式下CE必须保持高电平
4. **载荷宽度不一致**：发送端和接收端的payload_width必须相同

**调试方法：**
```c
// 检查FIFO状态
uint8_t fifo_status = NRF_ReadReg(NRF_REG_FIFO_STATUS);
printf("FIFO_STATUS: 0x%02X\r\n", fifo_status);

// 检查CONFIG寄存器
uint8_t config = NRF_ReadReg(NRF_REG_CONFIG);
printf("CONFIG: 0x%02X (PRIM_RX=%u)\r\n", config, config & 0x01);
```

### 8.4 通信不稳定，时好时坏

**可能原因：**
1. **电源不稳定**：加大电容（100uF）
2. **射频干扰**：
   - 附近有WiFi路由器（2.4GHz）
   - 更换频道（避开WiFi频道）
3. **天线问题**：检查天线是否损坏
4. **距离过远**：提高发射功率或缩短距离

**优化方法：**
```c
// 1. 降低数据率（提高可靠性）
cfg.data_rate = NRF_DR_250KBPS;

// 2. 提高发射功率
cfg.tx_power = NRF_PWR_0DBM;

// 3. 增加重传次数
NRF_WriteReg(NRF_REG_SETUP_RETR, (0x0F << 4) | 0x0F);  // 重传15次
```

### 8.5 如何实现多对一通信（多个发送端，一个接收端）

**方案1：时分复用**
- 每个发送端在不同时间片发送
- 接收端轮询接收

**方案2：使用不同频道**
- 每个发送端使用不同频道
- 接收端快速切换频道接收

**方案3：使用不同地址**
- 接收端使用多个Pipe（Pipe0~Pipe5）
- 每个发送端对应一个Pipe地址

## 9. 总结

### 9.1 关键要点
1. **地址配置是关键**：发送端的TX_ADDR = 接收端的RX_ADDR_P1
2. **参数必须一致**：频道、数据率、地址宽度、载荷宽度
3. **模式切换**：发送前切TX模式，接收前切RX模式
4. **RX模式下CE必须高电平**：否则无法接收

### 9.2 推荐配置
```c
// 稳定可靠的配置
cfg.channel = 40;                // 避开WiFi频道
cfg.data_rate = NRF_DR_250KBPS;  // 低速率，高可靠性
cfg.tx_power = NRF_PWR_0DBM;     // 最大功率
cfg.addr_width = 5;              // 5字节地址
cfg.payload_width = 32;          // 32字节载荷
```
